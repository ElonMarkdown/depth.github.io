一、确定托管代码的平台
GIthub
对于涉世不深的小白，由于网络传输速度的限制，其浏览器一般无法快速打开国外网站；因此需下载加速器 -- 网易UU加速器，下载完之后，打开软件，搜索学术资源，然后点击开始加速，随后会自动弹出一个网站，这个时候不用管他，直接新建页面搜索GitHub，这时加速器可加快你访问外网的速度。注意不使用UU加速器的时侯不要忘了停止加速，不然它会占用你的CPU资源哦。

二、注册Github账号并创建仓库
1、注册Github账号
如果你没有Github的账号，那么可以进入官网开始注册（注意一下用户名的填写，如果不使用自定义域名，用户名将会是你的Github分配给你的域名，比如你的用户名为xxx，那么你的域名会是xxx.github,io）。它为全英文界面，你可以下载英文翻译的插件滑动翻译或者复制粘贴到翻译软件上变成中文来理解，注意注册的时候有个奇葩要求 - - 根据它的要求选择相应的图片判断你是人还是机器，你看懂题目要求后非常简单。

Github官网：https://github.com
2、新建仓库
如图所示，点击右上角的“+”号，然后点击“New respository”即可。

3、配置仓库
这里推荐仓库名填写格式为：”用户名.github.io“。

然后点击”Add a README file“，再点击”Create repository“即可。

4、点击仓库的”Settings“，进入”pages“。


你就会看到你的域名已经正常显示出来了

三、创建Github token
1、developer settings
点击右上角头像的settings，选择菜单最下面的developer settings

2、Generate new token
点击”Personal acces tokens“，再点击”Generate new token“。

3、New personal access token
Note备注可以随便写，这里写个”Griddea“，然后选择”No expiration“，再把”repo“打上✓。

然后下拉页面，选择”Generate token“创建token。

4、复制token
记得保存好，因为只显示一次，忘了又得重新申请。

四、配置Gridea
准备工作都完成了接下来是配置Gridea。

Gridea官网： https://gridea.dev

1、下载Gridea
进入官网，根据你的电脑系统下载好Gridea客户端，这里以windows为例。

2、配置Gridea
第一次使用他需要你进行应用的初始化配置，才能让他和 Github Pages 连接。配置很简单，可以参考下面的方法进行配置：
域名：Github Pages 对应的域名（例如：https://brick713.github.io 或 http://moyu.life）
仓库：你的静态文件存放的仓库（例如：http://Brick713.github.io）
分支：你的 Github Pages 对应的分支（例如：master）
用户名：仓库所属用户的用户名（通常就是你的 Github 账号的昵称）
邮箱：Git 推送时使用的邮箱（通常就是你的 Github 账号的邮箱）
Token: Git 推送时需要的 Token，用来向 Github 提交构建后的文件 Github personal access tokens
CNAME: 可通过这个选项配置你自己的域名（例如：moyu.life）
安装好后，打开Gridea的”远程“配置你的Github信息，然后保存，如下图。

然后点击左下角的检测远程链接，如果配置没问题，那就会显示远程连接成功。

如果连接失败，还请回到上述步骤自行检查。

3、发布文章。
点击文章，点击右上角的”+“号即可。

编辑完成后，点击右上角保存即可。

4、预览和同步
点击预览，本地预览后没问题即可点击”同步“推送至github。

注意预览之后一定要同步，即把自己写的以及保存的内容同步到自己建立的网站上，否则即便保存了文件以及配置，打开网页也无法显示Gridea上的内容

当然，你还可以继续配置其他信息，比如自己的网站名字，头像，页脚，图标等等。

写作支持markdown格式以及Katex公式
